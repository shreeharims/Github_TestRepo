##  Docker Volumes & Port Mapping — Quick Revision

###  Part A: Port Mapping

```
docker run -d nginx
```
 Runs Nginx container in background, but  not accessible from browser (no port mapping)

```
docker run -d -p 8080:80 nginx
```
Maps host port 8080 → container port 80, so you can access Nginx via browser

 Open:
`http://localhost:8080` → Nginx page visible
---

### Part B: Docker Volumes (Persistence)

```
docker volume create myvolume
```

 Creates a persistent storage volume named `myvolume`

```
docker volume ls
```

Lists all volumes (verify `myvolume` exists)

```
docker run -it -v myvolume:/data ubuntu bash
```

 Runs container and mounts volume at `/data`

```
cd /data
echo "Docker Volume Persistence Test" > test.txt
exit
```
 Creates a file inside volume and exits container

```
docker run -it -v myvolume:/data ubuntu bash
```

 Runs new container with same volume

```
cat /data/test.txt
```

 File still exists → proves data persistence 

---

###  Part C: Combined (Port + Volume)

```
docker run -d -p 8081:80 -v myvolume:/usr/share/nginx/html nginx
```

Runs Nginx with:

* Port mapping (8081)
* Volume as web root folder

```
docker run -it -v myvolume:/data ubuntu bash
echo "<h1>Persistent Web Page</h1>" > /data/index.html
exit
```

 Creates HTML file inside volume

Open:
`http://localhost:8081` → Shows persistent webpage

---
